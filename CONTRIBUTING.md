# Contributing to displaycommitversion

Thank you for your interest in contributing to `displaycommitversion`! We welcome all kinds of contributions, including bug fixes, new features, and documentation improvements.

To ensure a smooth review process, please follow these guidelines.

## 🚀 How to Contribute

1. Find or Create an IssueBefore making any code changes, please check the existing issues or create a new one (Bug Report or Feature Request) to discuss the changes you want to make.
2. Fork and Clone the RepositoryFork this repository to your own GitHub account and clone it locally:

```bash
git clone https://github.com/YOUR_USERNAME/displaycommitversion.git
cd displaycommitversion
```

3. Create a BranchCreate a new branch for your work. Use a descriptive branch name:

```bash
# For bug fixes
git checkout -b fix/issue-description

# For new features
git checkout -b feat/feature-name
```

4. Make Changes and TestMake your code changes. Please ensure that:

- Your code follows the existing project style.
- You test your changes to make sure they don't break existing functionality.
- (Optional)Run `npm run test` if tests are available.

5. Commit Your ChangesWrite clear and concise commit messages. We recommend following the Conventional Commits format:

```bash
git commit -m "fix: resolve issue with fetching git hash in CI"
git commit -m "feat: add custom formatting option"
```

6. Push and Open a Pull RequestPush your branch to your forked repository and open a Pull Request (PR) against the `main` branch of the original repository.

## 📋 Pull Request Checklist

When opening a PR, please ensure the following:

- [ ] The PR is linked to an existing issue.
- [ ] The code builds successfully without errors.
- [ ] Commit messages follow the project's formatting guidelines.
- [ ] You have updated the documentation (like `README.md`) if your changes introduce new features or change configuration options.

## 💬 Code of Conduct

Please be respectful and collaborative when communicating within this project. Thank you for making `displaycommitversion` better!
